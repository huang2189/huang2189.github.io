關聯式資料庫管理系統(RDBMS)

  • 1970年由Dr.E.F.Codd提出的關聯式資料模型發展出來的資料庫管理系統。
  • 關聯式資料模型主要組成：
    關聯式資料模型 關聯式資料庫管理系統
    物件或關聯的集合(Collection of objects or relations) 資料庫物件(Database Objects)
    關聯運算方法(Set of operators to act on the relations) 資料運算(Data Operations)
    資料整合條件--正確性和一致性(Data integrity for accuracy and consistency) 資料庫限制條件--(Database Constraints)

關聯式資料庫

由二維矩陣的資料表(table)的集合所組成

資料表(Table)

  • 關聯式資料庫中最主要的資料庫物件
    • 資料儲存在資料表中
  • 欄位(Column)
    • 每一個欄位都必須 命名而且必須設定 資料型態 和 資料長度，用來存放欄位資料。
  • 資料列(Row)
    • 資料列是資料表中的一筆記錄資料

資料庫組成

Bit -> Byte -> Field -> Record -> File -> Database
Bit -> Byte -> Column -> Row -> Table -> Database

關聯資料庫的專有名詞

主鍵(Primary Key)

用於識別每一橫列的欄位組合，其特性如下：

  • 具有唯一性(Unique)
  • 必填，不為空值(Not Null)

直欄(Column)

為實體(Entity)或表格中的屬性，其特性如下：

  • 具有相同的資料型態
  • 儲存相似的內容資料

橫列(Row)

  • 由1 個以上的直欄所組成的一筆記錄(Record)
  • 作為SQL查詢所回傳的資料單位

欄位(Field)

  • 直欄與橫列交會的地方，作為使用者儲存資料的地方

空值(Null)

  • 欄位如果沒有儲存資料時稱空值

外來鍵(Foreign Key)

  • 欄位所輸入的資料必需出現在所參照的表格中主鍵或唯一鍵的鍵值
  • 欄位亦可為空值(預設為Null就不會檢查)
  • 每個外來鍵只能參考一個主鍵或唯一鍵

關聯式資料庫管理系統的功能

  • 儲存使用者資料
  • 提供安全與穩定的存取系統
  • 提供多人線上同時存取
  • 提供權限來做資料庫的管理
  • 提供資料備份與回復的功能
  • 支援結構化查詢語言

SQL的特性

  • SQL是高級的非程序的語言，它允許用戶在高層資料結構上工作。
  • 不要求使用者指定其資料的存放方法。
  • 不需要使用者暸解其具體的資料存放方式。
  • 不同底層結構的資料庫之間使用相同SQL語言作為資料的輸入與管理。
  • SQL語言可以寫出非常複雜的語句。

SQL語言的分類

  • 資料查詢語言(DQL：Data Query Language)--前端--
    • SELECT：查詢資料庫中表格的資料。
  • 資料操作語言(DML：Data Manipulation Language)--前端--
    • INSERT：可新增資料庫表格內的資料。
    • UPDATE：可更新資料庫表格內的資料。
    • DELETE：可刪除資料庫表格內的資料。
  • 資料定義語言(DDL：Data Definition Language)
    • CREATE：可建立資料庫中的物件。
    • ALTER：可變更資料庫中物件的結構。
    • DROP：可刪除資料庫中的物件。
    • RENAME：可變更資料庫中的物件名稱。
    • TRUNCATE：清除資料庫中表格內的資料。
  • 資料控制語言(DCL：Data Control Language)
    • GRANT：授予權限。
    • REVOKE：移除權限。

資料庫管理系統伺服器(DBMS Server)

作業

  • 安裝Oracle Server 及SQL Developer。

  • 使用Run SQL Command Line以系統管理者帳號(system)登入系統。

  • 建立一個新的使用者帳號。

  • 使用新的使用者帳號登入系統。

  • 加入新的資料庫。
